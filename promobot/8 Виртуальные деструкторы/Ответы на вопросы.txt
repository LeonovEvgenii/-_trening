1) Что такое виртуальный деструктор и зачем он нужен?

Определение - деструктор, помеченный словом virtual.
Находится в базовом классе по отношению к производному классу.

Нужен для обязательного вызоыва деструтора производного класса при удлаении объекта через указатель.


2) Что произойдет, если в базовом классе деструктор не будет виртуальным при удалении объекта через указатель на
 базовый класс?

При удалении объекта будет вызван только деструктор базового класса.
Деструктор производного не будет вызван.


3) В каких случаях обязательно использовать виртуальные деструкторы?

В случаях необходимости обзательного вызова деструктора производного класса.
При удалении объекта через указатель на базовый класс.


4) Как объявить чисто виртуальный деструктор и когда это может быть полезно?

Синтаксис

virtual ~name_class () = 0 ;

Объявляется в абстрактном классе.
Можно класс сделать абстрактным используя чистый виртуальный деструктор,
 если нет ни одного чистого виртуального метода.

Полезно может быть для вызова деструктора производного класса, 
если объект был создан с помощью указателя на базовый класс.

Так же предотвращает создание объекта класса, т к класс становится абстрактным, 
но позволяет работать с указателями на этот класс.
